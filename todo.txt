# Project Overview
This project provides utility functions with clear logic and easy-to-use interfaces.
The purpose of this context is:
1. To allow an LLM to understand the high-level functionality of files.
2. To provide structured summaries for potential future code agents.
3. To maintain a strict, minimalistic docstring summary for each file and function.

# File-Level Docstring Format
- File docstrings follow the format:
  -------- <file_name> --------
  File summary: <one-line summary of file>
  Function index:
  <function_name>:
      Summary: <one-line function description>
      Parameters: <param descriptions>
      Returns: <return description>
      Example: <example usage>
  -------- <file_name> --------

- Notes:
  * Only include classes if they exist.
  * Separators (----, ===, ~~~) are stripped.
  * Empty lines are ignored.
  * Function summaries are always one-line for LLM efficiency.
  * Full code is **not included** in the JSON summary but is stored separately for personal reading.

-------- test --------

# LLM Implementation Notes
1. Always use the top-level file docstring for summary extraction.
2. Each function should have a one-line summary; full docstring is optional but stored for personal reading.
3. Future extension: A coding agent can use this context to answer indepth questions about any function by referencing the stored full docstring.
4. No code blocks or actual implementation should be fed to the LLM in the JSON summary—only structured text for efficiency.
5. This context can be stored in vectors or embeddings later for semantic search or retrieval-based code assistance.

# Best Practices
- Keep function summaries concise.
- Only include classes when present to save token usage.
- Maintain consistent formatting with the wrapping separators.
- Use clean one-line summaries for LLM ingestion.



-------- Context: Project Overview --------

Modules:

1. cli.py
   Purpose: Orchestrates scanning, parsing, summarizing, and exporting project files.
   Functions:
     - process_project(project_path): Scan Python files, extract summaries, handle caching and export.

2. scanner.py
   Purpose: Project filesystem scanning and structure tree generation.
   Functions:
     - scan_project_files(path): Return list of Python files.
     - build_tree(path): Build hierarchical project file tree.

3. parser.py
   Purpose: Parse Python files, extract structured docstrings, functions, and classes.
   Functions:
     - parse_file(file_path): Extract module docstring, functions, and classes.
     - parse_docstring(module_doc): Convert module docstring to dict with file_summary and function index.
     - extract_function(node): Extract function name, arguments, return type, docstring from AST node.
     - extract_function_summaries(module_doc, parsed_functions): Map function names to one-line summaries.
     - format_docstring(name, doc): Clean and wrap a docstring consistently.
     - clean_docstring(doc): Remove empty lines, separators, fix bullets.

4. summarize.py
   Purpose: Prepare LLM-safe summaries, caching, and future LLM-based summarization.
   Functions:
     - summarize_file(file_path, module_doc): Placeholder for LLM summarization of new/modified files.
     - load_cache(): Load previously processed file summaries.
     - save_cache(cache): Save cache of summaries.

5. exporter.py
   Purpose: Output structured summaries and full docstrings.
   Functions:
     - export_json(data, filename, path): Write JSON summaries.
     - export_text(text, filename, path): Write full docstrings for human reading.

Notes:
- Docstring format wraps all docstrings with "-------- {name} --------" for readability.
- File summary is minimal, one-line for LLM feeding.
- Function summaries are extracted from the top docstring function index.
- Full docstring is kept separately for human-readable context.
- Future extension: vectorize function summaries and full docstrings for coding agent queries.
- New files or modified functions will trigger LLM summarization (placeholder implemented).
- Only include classes if they exist to save tokens.
